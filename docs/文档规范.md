# 项目文档规范

## 文档组织结构

### 文档存放位置
- **所有项目文档必须放置在 `docs/` 文件夹中**
- 唯一例外：`README.md` 保留在项目根目录（用于 GitHub 项目首页展示）

### 文档命名规范
- 使用有意义的文件名，清晰描述文档内容
- 中文文档优先使用中文文件名（如：`项目初稿设计.md`、`文档规范.md`）
- 英文文档使用英文文件名（如：`QUICKSTART.md`、`BUILD.md`）
- 中英文对照文档使用 `_zh` 后缀（如：`README_zh.md`、`CHANGELOG_zh.md`）

## 文档语言要求

### 主要语言
- **所有项目文档必须使用中文编写**
- 文档描述、说明、注释等内容必须使用中文

### 双语支持
- 重要文档（如 README、QUICKSTART）可提供英文版本
- 英文版本文件名不带后缀，中文版本使用 `_zh` 后缀
- 示例：
  - `README.md` - 英文版
  - `README_zh.md` - 中文版

## 文档类型

### 必需文档
1. **README.md** - 项目介绍（根目录）
2. **README_zh.md** - 项目介绍中文版（docs/）
3. **项目初稿设计.md** - 项目设计文档
4. **CHANGELOG_zh.md** - 更新日志
5. **文档规范.md** - 本文档

### 可选文档
- **QUICKSTART_zh.md** - 快速开始指南
- **BUILD.md** - 构建说明
- **IMPLEMENTATION.md** - 实现细节
- **功能说明.md** - 各类功能说明文档

## 文档内容规范

### 文档结构
1. 使用清晰的标题层级（# ## ### ####）
2. 每个文档开头必须有一级标题
3. 使用有序列表和无序列表组织内容
4. 代码示例使用代码块标记

### 代码示例
```javascript
// 代码示例必须包含中文注释
function example() {
  // 这是一个示例函数
  console.log('示例');
}
```

### 链接引用
- 文档间相互引用使用相对路径
- 示例：`[快速开始](./QUICKSTART_zh.md)`

## 文档维护

### 更新要求
- 代码功能变更时，必须同步更新相关文档
- 新增功能必须编写对应的功能说明文档
- 重大变更必须更新 CHANGELOG_zh.md

### 版本控制
- 所有文档纳入 Git 版本控制
- 文档变更需要提交清晰的 commit 信息

## 文档审查

### 质量标准
- 文档内容准确、完整
- 语言表达清晰、易懂
- 格式规范、统一
- 无错别字、语法错误
