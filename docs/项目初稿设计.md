### 项目名称：跨平台屏幕区域录制工具（Electron + FFmpeg）
```
项目目标：开发一款用于Windows和macOS的桌面应用，允许用户通过鼠标框选屏幕任意区域进行高清录制，并使用FFmpeg进行编码与压缩。
```

#### 一、 核心功能需求

区域选择：启动应用后，用户可通过鼠标拖拽框选屏幕上任意矩形区域作为录制范围。界面需清晰显示选区坐标和尺寸。

录制控制：提供简洁的控制按钮（开始/暂停/停止）。录制时应有视觉反馈（如边框闪烁、计时器）。

视频输出：

编码与压缩：使用FFmpeg将录制的原始视频流编码为MP4格式，并进行压缩优化。

分辨率预设：提供 “720p” 和 “1080p” 两种输出质量选项（即使选区小于该分辨率，也按此标准缩放/处理）。

存储设置：首次启动或用户操作时，可弹出目录选择对话框，让用户设置默认视频存储路径。该设置需持久化保存。

基础设置：提供简单设置界面，可切换分辨率模式、查看和修改存储目录。

#### 二、 技术栈与规格

框架：Electron（主进程 + 渲染进程）

屏幕捕获：使用Electron的 desktopCapturer API 或 ffmpeg-screen-recorder 等成熟模块。

视频处理：集成 FFmpeg（可捆绑静态二进制文件或检测系统环境）。

开发语言：前端使用 HTML/CSS/JavaScript (可搭配 React/Vue 等)，后端逻辑使用 Node.js。

打包工具：推荐 electron-builder，以生成 Windows (exe/msi) 和 macOS (dmg) 安装包。

#### 三、 开发前需明确的关键细节（请逐一回答）

区域选框样式：你希望选框的边框颜色、粗细、是否有半透明遮罩？是否有最小选区限制？

FFmpeg集成方式：是希望将FFmpeg二进制文件打包进应用（增大体积但可靠），还是要求用户自行安装（减小体积但增加复杂度）？

视频参数细节：

帧率：默认帧率设定为多少？（例如 30 fps）

编码器：使用 H.264 还是 HEVC (H.265)？这直接影响文件大小和兼容性。

音频：是否需要同时录制系统音频、麦克风，或两者混合？

应用界面风格：是否有具体的UI设计稿、配色参考，或希望采用极简的窗口控制栏？

异常处理：对于“权限被拒绝”、“存储空间不足”等常见错误，你希望以何种方式提示用户？
